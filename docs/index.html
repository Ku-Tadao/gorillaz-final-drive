What It Was

Phase 1 web-era driving experiment

Final Drive put the band into the Geep and dropped players into a limited 3D environment inspired by the 19-2000 world. It originally lived on Gorillaz.com and later appeared on official media releases.

Setup Guide

How to use the preservation tools

  1. Start with your own original files

    Use a legitimately obtained copy from an official source such as the original release media or archived personal backup.

  2. Drop in the launcher

    Place Play Gorillaz.bat beside gorillaz_final_drive.exe.

  3. Patch your local IML32 build

    Run the included PowerShell helper against your own IML32.dll to avoid the old Director memory crash path.

    powershell -ExecutionPolicy Bypass -File .\tools\patch_iml32.ps1 .\IML32.dll

  4. Keep the runtime layout intact

    The EXE, launcher, required runtime files, and the xtras folder all need to stay together.

  5. Launch through the batch file

    The launcher shortens TEMP paths, forces the most stable renderer path, disables the worst compatibility traps, and pins the game to a single CPU core.

History Snapshot

What made Final Drive memorable

Final Drive was built during the era when the Gorillaz website doubled as an experimental multimedia playground. The project used Macromedia Director and Shockwave with a 3D toolchain that let the team create a compact driving sim around the Geep and the 19-2000 aesthetic.

It shipped with a small launch UI, looping music, selectable color depth and resolution, and a compact 3D map with ramps, road sections, and scenery that felt halfway between a tech demo and a proper playable bonus game.

That browser-plugin era is long gone, but the game still matters as one of the more tangible Phase 1 Gorillaz interactive projects. This repo exists to make the surviving Windows build easier to document and preserve.
